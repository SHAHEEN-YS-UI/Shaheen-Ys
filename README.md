## Overview

SHAHEEN-YS-UI is a feature-rich, self-hosted AI platform based on Open WebUI. It combines:

- A SvelteKit frontend (interactive UI, editors, collaborative features)
- A FastAPI backend (model orchestration, REST & WebSocket APIs)
- Optional integrations for Ollama, OpenAI, local transformers, and multiple vector stores (Chroma, Qdrant, Weaviate, Pinecone, Milvus, Valkey)
- Docker-first packaging with multi-stage Dockerfile and multiple docker-compose overlays (GPU, OTEL, Playwright)

This README consolidates installation instructions, environment variables, project structure, Docker configuration, and documentation found in the repository.

---

## Features (preserved & verified)

- SvelteKit-based responsive UI with dark mode
- FastAPI backend with OpenAPI docs and health endpoint
- Socket.io / WebSocket streaming for real-time interactions
- Multi-backend model orchestration: OpenAI, Ollama, local HF/transformers
- RAG (Retrieval-Augmented Generation) with embeddings (sentence-transformers, BGE)
- Document ingestion: PDF, DOCX, PPTX, XLSX, plain text, HTML
- Whisper / faster-whisper transcription support
- Support for multiple vector databases (optional): Chroma, Qdrant, Weaviate, Pinecone, Milvus, Valkey
- Docker and Docker Compose configurations (gpu, otel, playwright, data)
- Playwright test overlay and test dependencies
- Telemetry opt-out (SCARF_NO_ANALYTICS, DO_NOT_TRACK)
- Non-root / permission hardening build options (OpenShift friendly)

## Docker & Compose (analyzed)

Dockerfile highlights (multi-stage):

- Frontend build stage: node:22-alpine, npm ci, npm run build
- Backend base: python:3.11-slim-bookworm
- Build args supported (exposed in Dockerfile):
  - USE_CUDA (default false)
  - USE_CUDA_VER (default cu128)
  - USE_OLLAMA (default false)
  - USE_SLIM (default false)
  - USE_PERMISSION_HARDENING (default false)
  - UID, GID (runtime user; default 0)
  - USE_EMBEDDING_MODEL (default sentence-transformers/all-MiniLM-L6-v2)
  - USE_RERANKING_MODEL, USE_AUXILIARY_EMBEDDING_MODEL
  - USE_TIKTOKEN_ENCODING_NAME (default cl100k_base)
- Healthcheck: curl http://localhost:${PORT:-8080}/health | jq ...
- Default CMD: bash start.sh (backend/start.sh is the container entrypoint script)

- OLLAMA_BASE_URL="/ollama"
- OPENAI_API_BASE_URL=""

Docker Compose files present in repo:
- docker-compose.yaml (base)
- docker-compose.gpu.yaml (GPU overlay)
- docker-compose.otel.yaml (otel/tracing overlay)

Common ports and volumes:
- Exposes backend on port 8080 by default
- Volumes recommended for model caches and persistent vector DB storage (see compose files for exact mappings)

Example Docker build & run (CPU / default):
```bash
docker build -t shaheen-ys-ui:latest .
docker run --rm -p 8080:8080 \
  -e OPENAI_API_KEY="sk-..." \
  -e OLLAMA_BASE_URL="http://localhost:11434" \
```

## Installation

The repository contains both backend (Python/FastAPI) and frontend (SvelteKit) components. Below are verified installation steps, preserving the repository's instructions.

### Prerequisites

- Git
- Docker & docker-compose (recommended for production)
- Python 3.11
- Node 18+ (for frontend dev/build)
- npm or pnpm

### Clone repository
```bash
git clone https://github.com/SHAHEEN-YS-UI/Shaheen-Ys.git
cd Shaheen-Ys
```

### Python (venv) — backend (development)
```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r backend/requirements.txt
```

### Frontend (development)
```bash
# run in repository root if package.json references frontend (or inside src/ if applicable)
npm ci
npm run dev           # starts SvelteKit dev server (HMR)
# or build:
npm run build
```

### Run backend locally (development)
```bash
# from backend
bash backend/start.sh
# or directly:
uvicorn shaheen_ys_ui.main:app --reload --port 8080
```

### Docker (recommended for production)
```bash
docker build -t shaheen-ys-ui:latest .
docker run --rm -p 8080:8080 \
  -e OPENAI_API_KEY="sk-..." \
  -e OLLAMA_BASE_URL="http://localhost:11434" \
  shaheen-ys-ui:latest
```

### Docker Compose
```bash
# base compose
docker compose -f docker-compose.yaml up --build

# GPU overlay
docker compose -f docker-compose.yaml -f docker-compose.gpu.yaml up --build

# Playwright overlay (browser tests)
docker compose -f docker-compose.yaml -f docker-compose.playwright.yaml up --build
```

### Podman
Replace `docker` / `docker compose` with `podman` equivalents; ensure volumes & rootless configuration match expected UID/GID.

### Kubernetes / Helm
No official Helm chart included. Use the Docker image produced by Dockerfile. Provide ConfigMap/Secret for environment variables (WEBUI_SECRET_KEY, OPENAI_API_KEY). For OpenShift, enable permission hardening build arg (`USE_PERMISSION_HARDENING=true`) and run non-root user (set UID/GID).

### Windows
Use WSL2 for development or run backend `backend\start_windows.bat`. Use PowerShell for Node commands if not in WSL.

### Offline installation
Build image with pre-downloaded models or mount model cache volumes. Use `USE_SLIM=false` during build if you want the image to pre-download models (note: increases image size).

## How to Run (commands verified)

### Development workflow (frontend + backend)
1. Start frontend dev server:
```bash
npm run dev
```
2. Start backend in development:
```bash
source .venv/bin/activate
bash backend/start.sh
# or
uvicorn shaheen_ys_ui.main:app --reload --port 8080
```
3. Access UI at frontend port (SvelteKit) or backend if built & served.

### Production
Build frontend, then build Docker image which bundles the frontend in /app/build and serves via FastAPI:
```bash
npm run build
docker build -t shaheen-ys-ui:latest .
docker run -p 8080:8080 -e OPENAI_API_KEY="sk-..." shaheen-ys-ui:latest
```

### Common debug & maintenance commands
- View container logs:
```bash
docker logs -f <container-id>
```
- Check health endpoint:
```bash
curl http://localhost:8080/health
```
(The Dockerfile includes a HEALTHCHECK that expects /health to return JSON with status true.)
